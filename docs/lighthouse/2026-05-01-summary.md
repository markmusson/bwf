# Lighthouse audit — 2026-05-01

Run via `npx lighthouse … --form-factor=mobile --throttling-method=simulate`
against the prod build on `localhost:3100`. Raw HTML/JSON reports are
gitignored (see `docs/lighthouse/*.html` / `*.json`); regenerate with:

```
PORT=3100 npm start &
npx lighthouse http://localhost:3100/stadium \
  --output=json --output=html \
  --output-path=docs/lighthouse/stadium \
  --form-factor=mobile --throttling-method=simulate \
  --only-categories=performance,accessibility,best-practices,seo \
  --chrome-flags="--headless --no-sandbox"
```

- **Stadium hits Perf + A11y + SEO targets.**
- **Best Practices 77 on /stadium** is gated on two facts of life:
  1. **Third-party cookies** — flagged because the Stripe Embedded Checkout
     iframe sets cookies. We cannot opt out without removing Stripe.
  2. **Inspector issues** — generic "issues panel" entries from
     browser-console deprecations (most are coming from the
     `@convex-dev/auth` cookie path on initial sign-in checks).
  Both are upstream constraints. Score will recover if Stripe migrates
  to redirect-based checkout (out of scope) and Convex Auth catches up
  with the Chrome cookie partition deprecation.

## Wall is the cleanest page

`/wall` is purely Convex queries + DOM. Performance is 93 because there's
no canvas / no embedded iframe / no payment widget on the page. This
gives a useful upper-bound: when the user is past the donate flow, the
app is fast.
