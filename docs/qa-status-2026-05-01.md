## 2. Vitest + convex-test (in-process backend)

47 files / 319 tests, all green. Notable coverage:

- **Concurrency** — `convex/holds.test.ts`: two simultaneous claims for same seat → one wins, one gets `seat_held`. (Acceptance criterion #2 / #13)
- **Webhook idempotency** — `convex/donations.test.ts`: same `checkout.session.completed` twice → one paid update, one event row. (Acceptance #3 / #14)
- **Per-seat minimum** — premium £50 seat rejects £25 amount; standard £25 rejects £24; floor honoured at both `_createDraftForTest` and `stripe.createSession`.
- **Find-or-create user at markPaid** — Stripe customer email flows into `users` table; existing user reused on case-insensitive email match.
- **Multi-claim** — already-taken seats can be claimed again (`holds._claimSeatForTest`); `markPaid` increments `claimedCount`.
- **Tribute moderation** — clean → `approved`; profane → `pending`; admin approve/reject writes `adminAuditLog`.
- **Prize draw** — opt-in works via auth OR clientHoldId; `runDraw` deterministic from a seed, idempotent on `drawName`, includes both online + postal entries.
- **Rate limit** — fixed-window counter on `createDraft`/`holdClaim`/`tributeUpdate`/`prizeOptIn`; capacity proven via integration test.
- **Admin gate** — 6 cases on `requireAdmin`: unauthenticated, no-email, off-list, on-list, case-insensitive, empty `ADMIN_EMAILS`.

### Two data quality issues found

1. **No raglan seats in the DB.** STANDS now defines 7 stands; the live
   deployment was seeded against the old 6-stand layout, so the raglan
   north-west arc is empty. Visually fine because the canvas renders
   from `buildAllSeats(STANDS)`, but **clicking a raglan seat will fail
   on `holds.claim`** because `seats.list` won't have a matching DB row
   for `idByKey` lookup.
   - **Fix:** `npx convex run seats:seedSeats` (idempotent — only inserts
     missing rows, no destructive change).

2. **2 west-stand seats have `status="taken"` but `claimedCount=0`.**
   Pre-multi-claim donations (before commit `fb2461f`) flipped status
   without bumping the count. They render correctly today (the canvas
   `claimedCount ?? (status==="taken" ? 1 : 0)` fallback kicks in), but
   admin dashboard counts will skew by 2.
   - **Fix:** Backfill mutation that sets `claimedCount = 1` wherever
     `status === "taken"` and `claimedCount` is undefined. Trivial
     internalMutation; can be added on request.
